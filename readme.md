# 项目简介

本项目面向无先验地图条件下轮足机器人的长距离自主导航，研究对象为搭载三维激光雷达、IMU 和边缘计算平台的 Unitree Go2W 机器人。

系统利用 FAST-LIO2 提供机器人位姿估计与点云运动补偿，在机器人周围实时构建局部环境表示，包括：

- 地面高度
- 顶部障碍高度
- 垂直净空信息

该表示用于描述台阶、坡道、坑洼、粗糙地面、普通障碍及低矮通道等复杂场景。

## 技术路线

当前系统结构如下：

1. 传感器输入
   - XT16 激光雷达
   - IMU
2. FAST-LIO2
   - 位姿估计
   - 点云去畸变
   - 多帧点云对齐
3. ROG-Map
   - 局部三维占据地图构建
4. 双层高程—净空投影
   - 生成地面高度、顶部高度、净空和阻塞信息
5. Grid Map 发布
   - 多层局部环境信息输出

## 当前进展

- 已具备定位功能
- 需要完成的核心模块：局部环境表示
- 目标是形成可用于局部规划器的稳定环境输入

## 环境表示策略

对于学习型局部规划器，环境输入可以采用：

- 双层高程—净空地图
- 最近多帧局部点云
- 三维占据地图

直接使用连续点云存在以下问题：

- 不能简单堆叠，必须利用 FAST-LIO2 位姿将历史点云统一变换到当前机器人坐标系
- 需要固定范围裁剪和统一尺度编码
- 点数不固定，计算量大
- 自由空间与未知区域难以区分

因此，更合理的三维输入形式是：

- 由多帧点云生成的局部稀疏占据地图
- 明确表示占据、自由和未知空间
- 可用于稀疏三维网络特征提取

## 双层高程—净空地图优势

双层高程—净空地图将三维环境压缩为规则二维通道：

- 地面高度
- 顶部高度
- 净空
- 观测有效性

优点：

- 输入尺寸固定
- 物理含义清晰
- 便于可视化调试
- 训练和端侧部署难度较低

相比之下，稀疏三维占据地图：

- 保留墙体、台阶、顶部障碍和多层空间结构
- 但模型规模、数据量和计算资源要求更高

## 推荐阶段性方案

第一阶段建议采用：

- 多帧点云对齐
- ROG-Map 三维占据地图
- 双层高程净空投影
- 时序规划网络

优先完成稳定可部署的局部规划系统。

后续可增加稀疏三维占据分支，比较：

- 规则化二维表示与完整三维表示
- 特殊地形识别能力
- 规划成功率
- 推理速度
- Jetson 部署性能
